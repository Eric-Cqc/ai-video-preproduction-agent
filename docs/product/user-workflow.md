# 用户工作流

1. 创建项目并录入简报、目标、受众、约束与参考资料。
2. 系统或用户生成结构化草案；AI 输入与输出来源被记录，但草案仍可编辑。
3. 用户迭代叙事、脚本、场景、镜头和资产计划，形成制作蓝图版本。
4. 评审者评论、要求修改并批准指定版本；批准不会覆盖历史版本。
5. 用户导出经批准的交接包，供下游制作工具或团队执行。

## 冻结决定

人工批准是从“草案”变为“可交接”的明确状态转换；异步生成、导入和导出由可追踪 Job 执行（[ADR-007](../adr/ADR-007-background-jobs.md)）。

## 可替换假设与复审触发

角色名称、评审层级和导出载体可替换。当试点显示单人流与团队流需要不同的状态机，且已有至少三个真实项目证据时，复审工作流状态模型。
